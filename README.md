## 1. 概览

SGABM-MarketDynamics-Engine 是一个先进的多主体仿真（ABM）平台，旨在模拟“双碳”目标下，绿电市场中政策干预（供给侧财税优惠与需求侧绿电交易政策）、发电企业策略与用电企业行为在长期尺度下的协同演化过程 。 

该引擎已实现从数据依赖型脚本向**零依赖、参数驱动型**引擎的转变，研究人员可以直接通过参数调优探索政策协同效应、多主体互动与市场均衡。

## 2. 核心原理

本引擎的仿真逻辑建立在两大相互支撑的数学框架之上，通过“政策学习-市场博弈”的双层动态交互，共同模拟绿电市场的复杂演化过程与政策协同机制。

### 1. **三阶段 Stackelberg 序贯博弈** ：

   本引擎采用三阶段Stackelberg序贯博弈来构建市场运行的微观核心。在该框架中，绿电企业作为拥有先动优势的“领导者”，而用电企业则作为根据领导者信号进行反应的“跟随者”，政府则作为设定博弈规则（政策）并从中学习的外部调节者。这精准刻画了现实中政策先行、发电企业投资、用电企业消费的决策时序与权力结构。
   
#### 第一阶段（绿电企业初始决策）：
在此阶段，作为市场领导者的绿电企业，基于对政府当期财税优惠政策（如所得税减免、投资补贴）的认知，以及对市场需求的前瞻性预期，以长期利润最大化为目标，同时制定初步的绿电市场报价与对应的产能投资决策。该决策是在不确定性下的预判，旨在抢占市场先机并锁定产能布局。

#### 第二阶段（用电企业响应）：
在观测到绿电企业的初步报价信号后，作为跟随者的用电企业开始决策。它们在必须满足外生给定的电力总量需求与政府强制性的绿电消纳配额（RPS）双重约束下，以实现自身购电总成本最小化为目标，计算出最优的绿电与传统电力的采购组合。此过程本质上是一个受约束的线性优化问题，其解形成了对绿电的实际市场需求曲线。

#### 第三阶段（绿电最终定价与市场出清）：
绿电企业观测到由第二阶段汇总而来的实际市场需求后，将启动价格调整机制。它们基于贝叶斯更新思想，综合考虑自身生产成本、初始报价以及市场反馈的支付意愿，对初步报价进行修正，形成最终挂牌价格。随后，交易中心汇集所有绿电企业的最终供给曲线与用电企业的需求曲线，按照“价格优先、供需匹配”的原则进行统一出清，确定最终的绿电交易量、出清价格及结算方式，完成一个完整市场周期的闭环。

### **EWA (经验加权吸引力) 学习算法**：

为突破传统博弈论中政府“完全理性”的强假设，本引擎引入了行为博弈领域的经验加权吸引力（EWA）学习算法，用以模拟现实中政府的有限理性、适应性学习与政策演化的路径依赖特征。
在该算法框架下，政府每一项政策工具（如所得税优惠强度、消纳配额比例）的多个可选强度值都被视为一个“策略”。每个策略都有一个动态更新的“吸引力”值。每期仿真结束后，政府会计算当前政策组合带来的社会福利（涵盖碳减排效益、绿电交易量增长与政策财政成本）。EWA算法的核心在于，它不仅依据已实施策略实际产生的社会福利来更新其吸引力，还通过一个“想象力”参数，对未选择的策略可能带来的虚拟收益进行加权更新，模拟了政策制定者的“反事实思考”。
吸引力值按照特定的规则（涉及折旧率、学习强度等参数）进行更新后，政府在下一期选择某个政策强度的概率，由其吸引力的指数函数决定，形成一种“优势策略被更大概率选择，但仍有探索可能”的随机响应模式。这一过程使得政府的政策调整表现为一个渐进的、基于历史经验反馈的、并带有一定随机探索性的自适应学习过程，从而能够内生地演化出政策的调整路径与不同情景下的差异化政策组合。

## 3. 快速开始

### 3.1 环境准备
```bash
pip install -r requirements.txt
```

### 3.2 运行仿真
您可以直接运行覆盖所有默认情景 (S0-S6) 的批量仿真 。

```bash
# 默认运行（50年）
python run_simulation.py

# 自定义时长（如 20 年）
python run_simulation.py --years 20
```

### 3.3 数据可视化
生成他多维分析图表：

```bash
python visualize.py
```

## 5. 参数修改指南  
### 核心说明  
- 项目特性：已实现“**零依赖**”，所有初始状态均定义在 `core/base.py` 中。  
- 修改位置：唯一需调整的配置文件为 `core/base.py`。  

### 参数组详情  
按功能分类的参数组及关键说明如下：  

1. **政府偏好参数组**  
   - 关键常量：**GOV_ALPHA**、**GOV_BETA**、**GOV_GAMMA**  
   - 说明：社会福利函数中**环境、经济与政策成本**的决策权重（拟合值为 **4.44, 18.98, 5.2**）。  

2. **企业配置参数组**  
   - 关键常量：**GREEN_ENERGY_GROUPS**  
   - 说明：基于聚类分析划分的**低规模组**（中小型企业）与**中规模组**（头部企业）的初始资产、成本及产能设定。  

3. **用户配置参数组**  
   - 关键常量：**CONSUMER_GROUPS**  
   - 说明：基于自然垄断属性设定的**区域用电企业**，结合**GDP弹性系数**与**ARIMA模型**外生设定电力需求。  

4. **实验情景参数组**  
   - 关键常量：**SCENARIO_TARGETS**  
   - 说明：覆盖**S0-S6七大情景**（基准情景、强供给侧激励、强需求侧拉动、对称协同、区域差异化协同等）。  

5. **学习算法参数组**  
   - 关键常量：**EWA_PARAMETERS**  
   - 说明：政府自适应学习的核心参数（含**折旧率、想象力参数**等）。  


## 6. 输出解析  
仿真结果统一存储在 `batch_results/[时间戳]/` 目录下，核心输出分三类：  

### 1. 综合结果表（**comprehensive_results.xlsx**）  
涵盖**碳减排量、绿电渗透率、绿电累计交易量、出清价格、产能利用率、累计政策成本**等多维评估指标的详细时间序列数据。  

### 2. 总结报告（**summary_report.txt**）  
- 各政策情景的**综合绩效评估**与**排名**；  
- **最优政策协同路径**的推荐建议。  

### 3. 可视化文件夹（**visualizations/**）  
包含四类关键图表：  
- 关键指标**长期演化趋势折线图**；  
- 多政策情景**绩效对比雷达图**；  
- 主体交互**网络弦图**；  
- 政策工具**动态调用热力图**。  


## 7. 应用范围  
项目聚焦四大核心场景，支撑绿电市场政策设计与演化分析：  

### 1. 政策协同效应评估  
本仿真引擎的核心应用在于系统评估不同政策组合的协同效应。通过预设从S0（基准）到S6（区域差异化）的七类情景，引擎能够量化对比“强供给侧激励”、“强需求侧拉动”以及“对称协同”等政策组合的长期效果。分析聚焦于这些组合如何影响绿电市场的均衡状态，包括出清价格与交易量的形成；如何作用于经济运行，体现在绿电企业利润、产能利用率与用电企业成本的变化；以及最终如何在环境维度实现碳减排。仿真结果能够直观揭示，为何“对称协同”情景（S3）能比单一侧政策更优地实现经济增效与环境减排的“双重红利”，为政策制定者提供超越定性判断的量化证据。

### 2. 市场主体策略推演  
引擎精细刻画了市场主体的异质性及其策略互动。在供给侧，绿电企业被区分为“中规模组”（头部企业）和“低规模组”（中小企业），它们拥有不同的初始资产、成本结构和风险偏好，因此在面对相同的政策信号时，会做出差异化的投资与定价决策。在需求侧，用电企业则受到区域GDP增长与强制消纳配额（RPS）的双重约束。通过三阶段Stackelberg博弈框架，本工具能够模拟这些异质主体在政策压力、配额指标和电力成本波动下，如何从短期成本最小化响应，逐步演化为长期的战略投资调整，从而推演出微观主体行为如何汇聚并塑造宏观市场动态。  

### 3. 长期演化路径探究  
模型专为观察复杂系统的长期动态而设计。在跨越50个仿真周期（模拟年）的尺度上，引擎能够追踪关键指标的非线性轨迹。例如，可以观察到绿电出清价格如何从初始分化状态逐步收敛至一个由发电成本与替代价值决定的长期均衡点；绿电渗透率如何因投资滞后、政策学习与需求不确定性而持续呈现非周期性的震荡；以及累计政策成本如何在自我强化的正反馈机制下形成特定的增长路径。这些分析旨在超越静态均衡分析，揭示绿电市场内在的路径依赖、锁定效应以及作为复杂适应系统的演化特征。

### 4. 区域差异化设计  
引擎支持对“全国一盘棋”与“因地制宜”两种政策模式的效果测试。通过“区域差异化协同”情景（S6），用户可以模拟我国“西电东送”的典型格局：在西部可再生能源资源富集区，政策参数可设置为侧重供给侧激励（如更高的设备投资补贴），以鼓励发电投资；在东部负荷中心地区，则侧重需求侧拉动（如更高的消纳配额与消费补贴），以保障绿电消纳并激励本地用户。通过对比此类差异化设计与全国统一政策（如S3）在整体效率、区域公平性、跨区交易流量等方面的表现，可为制定更具精准性和适应性的“一区一策”提供模拟实验依据。
